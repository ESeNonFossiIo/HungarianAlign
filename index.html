Optimal sequence alignment,
solved as an assignment problem.

HungarianAlign pairs two labelled sequences by turning their co-occurrences into a cost matrix and finding the permutation that maximises its trace — the best overall alignment — using the classic Hungarian algorithm.

What it does

A small, transparent solver

Every label pairing increments a cell of a cost matrix; the solver finds the row/column permutation that maximises the trace. Each intermediate step is printed so you can watch the algorithm work.

Cost matrix from labels

SequenceProblem reads two label streams and tallies every observed row/column pairing into a Hungarian cost matrix.

Maximise → minimise

HMatrix flips the maximise-trace goal into its complement, then reduces rows and columns and covers zeros until an optimal assignment is reachable.

Optimal alignment

The final permutation is the best overall alignment of the two sequences — printed alongside every reduction step for full transparency.

How it works

From two sequences to one optimal pairing

The pipeline is intentionally readable — a handful of focused classes, each doing one job.

  1. Read & tallySequenceProblem pairs the two input files entry by entry and fills the cost matrix from observed label co-occurrences.
  2. ComplementHMatrix converts the maximise-trace problem into its minimise-cost form (the complement matrix).
  3. Reduce & coverRows and columns are reduced, then zeros are covered with the minimum number of lines until an optimal assignment exists.
  4. AssignMask, Permutation / LPermutation and the utilities namespace handle the bookkeeping and emit the final alignment.

Quick start

Build and run in four commands

Needs only a C++11 compiler and CMake (≥ 3.10). Doxygen is optional, for the docs.

build & run
# compile and align the example data
mkdir build && cd build
cmake ..
make
make run
tests & docs
# run the test suite
cmake .. -DENABLE_TESTS=ON
make && ctest -V

# build the API documentation
make doc

Input format

Each .dat file lists one entry per line as index label. The two files are paired entry by entry until one is exhausted.

data/example_Q6.dat data/example_Q7.dat
example.dat
# index  label
1  1
2  1
3  3
4  3
5  3
